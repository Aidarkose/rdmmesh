## 0. TL;DR

- **F4 delivery-IT** (закрывает E14.12 §3 #2). `EgressDeliveryIT extends
  PostgresIT` доказывает, что `EgressPolicy` дискриминирует адресата прямо
  в `WebhookDeliveryWorker.drainOnce()` (DNS-rebinding-safe точка):
  - **blocked**: подписка на `169.254.169.254` (cloud-metadata, hard-deny,
    allowlist пуст) → outbox `GIVE_UP: egress … link-local/cloud-metadata`
    **без retry** (`delivered_at` проставлен), `last_delivery_status=FAILED`,
    `attempts=1`;
  - **delivered**: подписка на site-local адрес хоста, явно внесённый в
    allowlist (`Cidr "<ip>/32"` — Q56-путь), на котором поднят реальный
    `HttpServer` → POST доставлен 2xx, payload получен **байт-в-байт**,
    `X-RDM-Event-Id` проставлен, `delivered_at` + `last_error IS NULL`,
    `last_delivery_status=OK`.
- **F3 chunked hard-cap** (закрывает E14.8 §3 #3). Servlet-уровневый
  `RequestBodyCapFilter` (регистрируется на `/*` раньше Jersey):
  fast-path по `Content-Length` (413 до Jersey) + оборачивание тела в
  `CappingInputStream` — рвёт чтение на `MAX_BODY_BYTES+1`-м байте
  **без `Content-Length`** (chunked). Overflow → `RequestBodyTooLargeException`
  → `413` через `RequestBodyTooLargeExceptionMapper` (единый контракт
  ошибки `{"error":…}` с `RequestSizeLimitFilter`). Лимит переиспользует
  `RequestSizeLimitFilter.MAX_BODY_BYTES` (1 MiB) — единый источник.
- **Тесты F3 без сервера**: `CappingInputStreamTest` (4) — лимит
  inclusive, byte-wise и bulk overflow, пустое тело. Гоняется локально.

## 2. Решения

- **Почему «приватный-в-allowlist», а не «публичный» для delivered-кейса.**
  E14.12 §3 #2 формулирует «публичный → доставка», но в герметичном IT
  нет публичного 2xx-приёмника, а loopback egress'ом hard-deny (сервер на
  `127.0.0.1` поднять нельзя). Site-local адрес хоста приватный (в
  `PRIVATE_RANGES`) → доставка туда **дополнительно** доказывает, что
  allowlist (единственное, что банк задаёт по Q56) реально открывает
  egress. Публичный-разрешён уже покрыт герметичным `EgressPolicyTest`
  (8/8, E14.12 §1) на уровне политики — в IT не дублируется.
- **`assumeTrue(siteLocal != null)`** в delivered-кейсе: на машине без
  site-local NIC кейс скипается (не ложный провал). На CI-ubuntu private
  NIC есть → кейс исполнится.
- **F3 — servlet-слой, не Jetty `httpConfiguration`.** В Jetty-core нет
  готового «max request content size» для не-multipart; счётчик
  прочитанных байт — единственное, что ловит chunked. Вынесен в чистый
  `CappingInputStream` → unit-тестируется без поднятия сервера (закрывает
  и E14.8 §3 #4 для F3).
- **Unchecked-исключение** (а не `IOException`): проходит сквозь
  JAX-RS `byte[]`-MessageBodyReader (OM-webhook `rawBody`) и попадает в
  ExceptionMapper, а не в невнятную 500.
- **isMatchAfter=false**: фильтр становится перед остальной цепочкой
  (включая Jersey-servlet `/api/v1/*`), покрывая
  `/api/v1/webhooks/om/ownership`.
