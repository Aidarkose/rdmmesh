## 0. TL;DR

- **CI-gating (главное).** Обнаружено: `ci.yml` push-job гонял
  `mvn -DskipITs verify` — IT **не исполнялись нигде** (E1 §3.7 «skip
  on first iteration» так и не сняли). Фикс: push-job → `mvn verify`
  (без `-DskipITs`). ubuntu-latest имеет нативный dockerd, Maven там на
  host-JVM (setup-java) → Testcontainers подключается к демону напрямую
  (ограничение Docker-Desktop-прокси E14.9 §2 на CI НЕ действует). IT
  теперь реально гоняются и **гейтят merge** в main/release.
- **`AtomicRollbackIT`** (R9-backlog §3 #2, закрывает E14.5 §4 known
  debt). Доказывает атомарность `WorkflowService.transition`:
  fault-trigger `RAISE` на `workflow.workflow_transition` → весь tx
  откатан (status DRAFT, нет journal, нет approval_task); снятие
  триггера → happy-path проходит целиком.
- **`ArchiveRoundtripIT`** (E14.11 §3 #2). Postgres + **RustFS
  Testcontainer** (`rustfs/rustfs:1.0.0-beta.3`): seed audit_log →
  `archiveMonth` → манифест + объект в RustFS (`exists`) +
  `verifySegment` (скачать, пересчитать SHA-256) + идемпотентность.
  Переводит ручной `mc`-smoke (E14.11 §5.2) в CI-гейт.

Все 5 IT (`Migrations`, `HashChainRoundtrip`, `V073PartitionRouting`,
`AtomicRollback`, `ArchiveRoundtrip`) — `extends PostgresIT`,
`disabledWithoutDocker=true` (локально Skipped, CI — run).

## 2. Проверка локально (2026-05-17)

```
./bin/mvn -DskipITs verify → BUILD SUCCESS (компиляция IT ок:
  WorkflowModule.build/.service(), AuditArchiveService, RustFsArchiveAdapter,
  GenericContainer/Wait — все API резолвятся; surefire зелёный; ArchUnit 11)

  AtomicRollbackIT 1 Skipped · ArchiveRoundtripIT 1 Skipped (как и
  Migrations/HashChain/V073 — disabledWithoutDocker, E14.9 §2)
```

Корректность IT обеспечена **чтением схем/сигнатур** (нельзя прогнать

(V030 — `version_id`/`closed_at`/`candidate_users`),
`WorkflowModule.build(jdbi,lifecycle,ownership,catalog,bus).service()`.

→ без ownership-сетапа). `ArchiveRoundtripIT` сеет в DEFAULT-партицию
(месяц 2027-09 без помесячной — safety-net; findExportPage фильтрует
по occurred_at).

## 3. Что осталось

1. **Первый push в main → проверить CI.** Реальный прогон IT — только
   на GitHub Actions. Убедиться: verify-job зелёный, 5 IT исполнились
   (не Skipped — на ubuntu dockerd есть). Если RustFS-контейнер флапает
   по готовности — ужесточить `Wait` (сейчас `forHttp("/").forStatusCode(403)`;
   запасной — `Wait.forListeningPort()` + retry на первом S3-вызове).
2. **Re-archive под GOVERNANCE.** `ArchiveRoundtripIT` повторно
   архивирует сегмент (ожидает `manifestInserted:false`). Если сборка
   RustFS на CI блокирует повторный PUT под object-lock иначе —
   скорректировать ассерт (поведение проверено руками E14.11, но
   версия RustFS на CI = та же 1.0.0-beta.3, риск низкий).
3. **Прочее без изменений:** Q56 (webhook CIDR-allowlist банка),
   Q62-финал (GOVERNANCE/COMPLIANCE prod), helm CronJob, V2-territory.

## 4. Итог по «нужен CI/dockerd»

Закрыто: оба IT написаны (корректны по чтению, компилируются,
failsafe-discovered) + CI-gating включён. Теперь это **не «отложено
из-за окружения»**, а «исполняется на CI» — локальный Docker-Desktop
по-прежнему скипает (by design, E14.9 §2), но merge в main гейтится
реальным прогоном на GitHub Actions.
