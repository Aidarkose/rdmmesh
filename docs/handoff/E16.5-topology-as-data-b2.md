## 0. TL;DR

- **BPMN кодирует граф.** Process-extension
  `<rdm:workflowGraph>JSON-рёбер</rdm:workflowGraph>` (поля
  from,to,action,kind[,reject,recordReviewer,setApprover]).
  `WorkflowGraphCodec` ⇄ JSON; нет элемента → домен на дефолтном 4-eyes
  (обратная совместимость с шаблонами round 2).
- **Deploy-time gate.** `BpmnTemplateValidator.validate` парсит граф и
  ОБЯЗАТЕЛЬНО прогоняет `WorkflowGraphInvariants` (B1-сеть) ДО Flowable-
  деплоя: non-compliant → `IllegalArgumentException` → 400, ничего не
  задеплоено/не записано. Канонический JSON → `workflow_template.graph_json`
  (Flyway **V033**).
- **Runtime — граф домена авторитетен.** `WorkflowService.transition`:
  `resolveGraph(codeSet.domainId())` → активный `graph_json` домена →
  `StateMachine.validate(req, graph)` + `graph.nextRequiredRole(to)`.
  Нет шаблона/NULL/битый/не прошёл инварианты → **fail-safe** дефолтный
  4-eyes (строжайший known-good — НЕ ослабление). Инварианты
  **re-validated на чтении** (defense-in-depth: подмена DB-row не
  ослабит no-bypass в рантайме).
- **Доказано (IT):** домен с графом без `owner_reject` →
  `STEWARD_APPROVED→DRAFT` (легален в дефолте) даёт
  `IllegalStateTransition`, а `owner_approve` проходит → судит граф
  ДОМЕНА. Non-compliant граф в BPMN → 400.

## 2. Совместимость / безопасность

- **Нулевая регрессия:** домен без шаблона/без графа → `resolveGraph` →
  `defaultFourEyes()` == прежняя матрица (StateMachineTest 22/22, прочие
  IT не изменились). enum-движок (нет шаблонов) — дефолт всегда.
- **No-bypass для кастомных графов** держится на двух рубежах:
  (1) deploy-time `WorkflowGraphInvariants` (нельзя задеплоить граф в
  обход STEWARD→OWNER); (2) runtime re-validate в `resolveGraph` (битый/
  tampered → fail-safe дефолт). Плюс runtime-guard'ы STEWARD/OWNER
  (B1) на рёбрах графа → 3 разных лица.
- Парс графа на каждом переходе (граф крошечный). Кэш по
  (domain,version) — возможный V1+-debt (не делал — преждевременно).
